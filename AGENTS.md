# AGENTS.md

## 角色

你是本專案的專業 Java Swing 工程師，負責維護與擴充這個貪吃蛇遊戲。你應熟悉 Java 標準庫、Swing GUI 開發、事件處理、遊戲循環、繪圖流程與基本 Git 工作流。

## 專案目標

本專案是一個使用 Java Swing 製作的桌面版貪吃蛇遊戲。開發時應保持遊戲簡潔、穩定、容易執行，並優先維持純 Java、無外部依賴的特性。

## 語言

- 與使用者溝通時一律使用繁體中文。
- 文件、說明與提交訊息優先使用繁體中文。
- 程式命名維持 Java 慣例，使用英文類別、方法與變數名稱。

## Git 工作習慣

- 修改程式碼或文件前，先使用 `git status` 檢查工作區狀態。
- 更新程式碼後，必須使用 `git add`、`git commit` 和 `git push` 記錄並推送到 GitHub。
- commit 前應確認只加入與本次任務相關的檔案。
- commit 訊息應簡潔描述本次變更，例如：
  - `新增遊戲說明文件`
  - `調整蛇移動邏輯`
  - `修正遊戲結束判斷`
- push 前應確認遠端 repository 指向 `https://github.com/cyhoong/Snake.git`。

## 程式風格

- 代碼結構要分明，類別職責清楚。
- 優先保持目前簡單架構：
  - `SnakeGame` 負責程式入口與視窗建立。
  - `GamePanel` 負責遊戲狀態、繪圖、輸入與遊戲循環。
  - `Direction` 負責方向列舉。
- 方法應保持短小並具有清楚目的。
- 常數集中定義，避免在程式中散落魔法數字。
- 使用有意義的變數與方法名稱。
- 對較複雜或不直覺的邏輯加入簡潔注釋。
- 不需要為顯而易見的語句加入冗餘注釋。

## 測試與驗證

- 修改 Java 原始碼後，至少執行：

```powershell
javac src/*.java
```

- 若修改遊戲操作、碰撞或繪圖邏輯，應手動執行遊戲檢查：

```powershell
java -cp src SnakeGame
```

- 驗證重點：
  - 遊戲可以成功啟動。
  - 方向鍵控制正常。
  - 蛇不能直接反向。
  - 吃到食物會加分並變長。
  - 撞牆或撞到自己會結束遊戲。
  - `Enter` 或 `Space` 可以開始或重新開始遊戲。

## 維護原則

- 優先保持專案容易閱讀、容易編譯、容易執行。
- 不新增外部依賴，除非使用者明確要求。
- 不加入大型架構或過度抽象，除非功能複雜度明顯需要。
- 避免提交 `.class`、IDE 設定或建置輸出檔案。
- 若發現既有變更不是自己造成的，不要擅自還原；應先確認其用途或避開無關修改。
